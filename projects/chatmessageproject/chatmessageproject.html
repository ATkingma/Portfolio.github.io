Project Overview

This project highlights an implemtation of a realtime chat application built with ASP.NET Core and SignalR. It features user authentication, end to end encryption, and more for seamless communication across users. It demonstrates end-to-end encryption using RSA and AES, secure user authentication with JWT, and real-time messaging capabilities. The project serves as a comprehensive example of building a secure and responsive chat application using modern web technologies.

The project is build with zero trust security principles in mind, ensuring that all communication is encrypted and secure. It also includes a user-friendly interface built with Next.js, providing a seamless experience for users to connect and chat in real-time. Overall, this project showcases the power of ASP.NET Core and SignalR for building robust and secure real-time applications.

Architecture Decisions

The project is split between two segments frontend and backend. The frontend is built with Next.js, a React framework that provides server-side rendering and static site generation capabilities The backend is built with ASP.NET with multiple projects, one for the API and one for the SignalR hub. The API project handles user authentication, message storage, and other business logic, while the SignalR hub manages real-time communication between clients. This separation of concerns allows for better scalability and maintainability, as each segment can be developed and deployed independently. Additionally, using Next.js for the frontend ensures a fast and responsive user experience, while ASP.NET provides a robust and secure backend for handling real-time messaging and data management.

Theory: How Collision Detection Works

AABB Collision

Rectangle (AABB) vs. Rectangle: Two rectangles collide if their projections on both the X and Y axes overlap. This is known as Axis-Aligned Bounding Box (AABB) collision detection. AABB is very fast because it only requires comparing the min and max values of the rectangles on each axis. It works best for rectangles that are not rotated.

PseudoCode ⮞ 

collision: just compare min/max on X and Y
  boolean aabbCollide(Rect a, Rect b) {
    return a.x < b.x + b.width &&
        a.x + a.width > b.x &&
        a.y < b.y + b.height &&
        a.y + a.height > b.y;
  }

SAT Collision

SAT (Separating Axis Theorem): SAT is a more advanced algorithm that can detect collisions between any convex polygons, including rotated rectangles. It works by projecting the shapes onto potential separating axes and checking for overlaps. If a separating axis is found where the projections do not overlap, the shapes are not colliding. SAT is more flexible but also more computationally expensive than AABB.

Pseudocode ⮞ 

collision:
  boolean satCollide(Polygon polyA, Polygon polyB) {
    for (Vector axis : getAxes(polyA, polyB)) {
      Projection projA = project(polyA, axis);
      Projection projB = project(polyB, axis);
      if (!projA.overlaps(projB)) {
        return false; // Found a separating axis
      }
    }
    return true; // No separating axis, collision!
  }

Circle Collision

Circle vs. Circle: Two circles collide if the distance between their centers is less than the sum of their radii. This is calculated using the Pythagorean theorem.

How SignalR is implemented

SignalR is a library for ASP.NET that allows server code to send asynchronous notifications to client-side web applications. It enables real-time communication between the server and clients, making it ideal for applications like chat apps, live dashboards, and gaming. SignalR abstracts away the complexities of real-time communication, providing a simple API for developers to push updates to connected clients instantly. It supports various transport methods (WebSockets, Server-Sent Events, Long Polling) and automatically falls back to the best available option based on the client's capabilities. With SignalR, developers can easily create interactive and responsive web applications that require real-time updates without the need for constant polling.

Inside my project SignalR is used to create a real-time chat application. The server-side code uses SignalR to manage connections and broadcast messages to all connected clients. When a user sends a message, the server receives it and uses SignalR to push that message to all other clients in real-time. This allows for instant communication between users without the need for page refreshes or manual polling. Additionally, SignalR can be used to handle user presence, allowing the application to show which users are currently online and available for chat. Overall, SignalR provides the necessary infrastructure for building a responsive and interactive chat application that delivers messages instantly to all users.
